The big idea

The Model Is Not The Whole Product

Most useful AI products do not ask a model to remember everything. They bring the right evidence to the model at the right moment.

Without retrieval

The model answers from its training, guesses, or admits it does not know. This can sound confident even when the answer is wrong.

With retrieval

The system searches a trusted document first, sends relevant excerpts to the model, and asks it to answer using that evidence.

Workshop approach

Use The Tested Workshop Code First

Everyone will start from the same working code so the room can focus on understanding the system, not debugging different versions.

Baseline first

We will run the tested workshop files end to end: extract the PDF, create chunks, embed them, and retrieve evidence from the local vector database.

  • ✓scripts/01_extract_pdf.py
  • ✓scripts/02_chunk_basic.py and scripts/03_chunk_section_aware.py
  • ✓scripts/04_embed_basic_chunks.py and scripts/07_embed_chunks_section_aware.py
  • ✓scripts/05_retrieve.py and scripts/06_retrieve_multi.py

Experiment second

Once the baseline works, we will change one thing at a time and compare what happens. This is where the product learning becomes visible.

  • ✓Change chunk size and overlap
  • ✓Compare basic and section-aware chunking
  • ✓Change the question wording
  • ✓Change how many chunks are retrieved

Participants do not need to write the whole pipeline from scratch. Their job is to run it, inspect it, understand each part, and then experiment with controlled changes.

The first job is confirming that Python can read text from the PDF. This script uses pypdf to open the report and preview extracted text.

python scripts/01_extract_pdf.py
If extraction is blank or scrambled, the AI product will fail before the model is even involved.
3 CUT

Create Basic Chunks

Chunking breaks a long document into smaller pieces the system can search. Basic chunking splits by size, which is easy to understand but can cut across document sections.

python scripts/02_chunk_basic.py

Output

This creates outputs/auspost_chunks.json. Basic chunks are useful for comparison, but they do not understand document sections.

4 EMB

Embed The Basic Chunks

An embedding is a numerical fingerprint of meaning. The script uses nomic-embed-text to embed each basic chunk.

python scripts/04_embed_basic_chunks.py

Teaching move

Use this after 02_chunk_basic.py. The retrieved results will show what the basic chunks make easy or hard to find.

5 GET

Test Basic Chunk Retrieval

This retrieves the most relevant chunks from the vector database for the question. We are testing retrieval quality first, before asking any model to write an answer.

python scripts/05_retrieve.py --question "What was Australia Post revenue in FY25?"

Look for evidence

The test is whether the retrieved text contains the answer, not whether the model can generate a polished response.

6 CUT

Create Section-Aware Chunks

This is the improved chunking step. It keeps page ranges, section labels, chunk IDs, and overlap.

python scripts/03_chunk_section_aware.py

Trade-off

Chunks that are too small lose context. Chunks that are too large can bury the answer. Section-aware chunks help preserve meaning.

7 EMB

Embed The Section-Aware Chunks

Turn the section-aware chunks into vectors so we can test whether this chunking strategy retrieves useful evidence.

python scripts/07_embed_chunks_section_aware.py

Output

This creates the local vector database in outputs/chroma_db/.

8 GET

Test Section-Aware Retrieval

Ask the same question again and compare whether the retrieved chunks are better evidence.

python scripts/05_retrieve.py --question "What was Australia Post revenue in FY25?"

Important

Retrieval quality controls answer quality. The model can only reason over what we give it.

9 SEM

Create Semantic Chunks

Semantic chunking uses nomic-embed-text to compare neighbouring text units and split when meaning changes.

python scripts/08_chunk_semantic.py

Output

This creates outputs/auspost_semantic_chunks.json. To embed those chunks, run python scripts/07_embed_chunks_section_aware.py --chunks outputs/auspost_semantic_chunks.json.

Four Ways To Chunk A Document

Chunking type How it works When it helps
SimpleSplits text by size.Fast baseline for comparison.
Section-awareRespects document sections, page ranges, IDs, and overlap.Better context for structured documents.
SemanticUses embeddings to detect meaning changes.Better topic boundaries when headings are not enough.
HierarchicalUses small chunks for matching and larger parent chunks for context.Production retrieval where precision and context both matter.
DIY Next H

Where This Goes Next

Hierarchical chunking uses small chunks for precise matching and larger parent chunks for context. We are not implementing it today.

Small chunks = better matching
Large parent chunks = better context

Production pattern

Today we compare basic, section-aware, and semantic chunks. Hierarchical chunking is the next production-grade idea after that comparison makes sense.

10 GET

Try Multi-Query Retrieval

One user question can be searched several ways. This script combines and ranks the best retrieved chunks.

python scripts/06_retrieve_multi.py

Reliability move

Multi-query retrieval helps when the first wording misses the evidence. It is still retrieval, not reasoning.

11 QA

Break The System On Purpose

Good product teams test where the system fails, not only where it works.

python scripts/05_retrieve.py --question "What was Australia Post revenue in FY25?"
python scripts/05_retrieve.py --question "What is the CEO's favourite movie?"
python scripts/05_retrieve.py --n-results 8
python scripts/03_chunk_section_aware.py --max-words 500 --overlap-words 80

Discussion

When it fails, ask whether the issue came from the document, chunking, embeddings, retrieval, prompting, or the model.

Product judgement

Failure Modes To Notice

These are the questions PMs and Designers should ask when an AI answer looks wrong.

Failure mode What it looks like Product question
Bad extractionText is missing, scrambled, or full of table noiseDo we need better parsing or source formatting?
Bad chunkingThe answer is split across chunksWhat chunk size fits this content?
Bad retrievalThe system fetches plausible but irrelevant evidenceHow do we measure retrieval quality?
Weak guardrailThe model answers beyond the evidenceShould the product say "I don't know" more often?
No citationUsers cannot check the sourceWhat proof does the user need to trust this?
LatencyThe answer is technically right but too slowWhere is the acceptable speed-quality trade-off?

Product judgement

PM Takeaways

Retrieval is not an implementation detail. It shapes trust, UX, cost, speed, quality, and accountability.

PMs Should Ask

  • ✓What sources are trusted?
  • ✓How fresh is the knowledge?
  • ✓Can users inspect citations?
  • ✓What should happen when evidence is weak?
  • ✓How will we evaluate answer quality?

Designers Should Notice

  • ✓Where confidence is implied
  • ✓How evidence is shown
  • ✓What "I don't know" feels like
  • ✓How users recover from a bad answer
  • ✓Whether the system earns trust over time
